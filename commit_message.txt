fix(ci): Solucionar fallos del pipeline de CI y definir configuración de Poetry

Este commit aborda múltiples problemas que impedían el correcto funcionamiento del pipeline de CI y la instalación del proyecto con Poetry:

1.  **Configuración de Poetry:** Se añade la sección `[[tool.poetry.packages]]` en `pyproject.toml` para definir explícitamente la ubicación del paquete (`crud_app`), solucionando el error "No file/folder found for package".
2.  **Documentación del Proyecto:** Se crea un archivo `README.md` básico para satisfacer el requisito de `poetry install` y proporcionar una descripción inicial del proyecto.
3.  **Estabilidad del CI:** Se corrige la configuración de `trivy-action` en `.github/workflows/ci.yml`, cambiando `@master` a la etiqueta de versión mayor `@0` para asegurar la estabilidad y resolver el error de "repository not found".

Estos cambios garantizan que el proyecto pueda ser instalado correctamente por Poetry y que el pipeline de CI se ejecute de manera exitosa y reproducible.
