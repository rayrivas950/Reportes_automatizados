Este commit aborda varios problemas y refactoriza la base de código para mejorar la fiabilidad de las pruebas y la corrección de la API.

Los cambios clave incluyen:
- **Modelos:** Se añadió `related_name` explícito para las claves foráneas en los modelos `Compra` y `Venta`. Se eliminaron los métodos `save()` problemáticos, delegando la lógica de negocio a las señales de Django.
- **Serializadores:** Se eliminó `depth=1` de `CompraSerializer` y `VentaSerializer` para permitir operaciones de escritura en campos relacionados.
- **Señales:** Se introdujo `crud_app/signals.py` con manejadores `post_save` y `post_delete` para `Compra` y `Venta` para actualizar atómicamente el stock del producto, registrados a través de `apps.py`.
- **Pruebas:** Se refactorizaron `APITests` para usar `setUp` para la configuración por prueba y `setUpClass` para la limpieza de la base de datos, asegurando el aislamiento de las pruebas y la resistencia a datos obsoletos. Se corrigió la inversión de URL para el endpoint de resumen.
- **Vistas y URLs:** Se refactorizó `ReporteSummaryView` a un `ViewSet` llamado `ReporteSummary` y se registró correctamente con el router de DRF. Se añadió la importación `models` faltante en `views.py`.
- **Gitignore:** Se creó un archivo `.gitignore` completo para evitar que se suban archivos no deseados.
